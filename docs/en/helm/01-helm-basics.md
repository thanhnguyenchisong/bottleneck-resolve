## Part 1: Helm basics

### What Helm is
- Package manager for Kubernetes: bundles manifests into **charts** with versioned releases.
- Separates **templates** (manifests) from **values** (config), enabling repeatable deploys.

### Install
- macOS: `brew install helm`
- Linux (script): `curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash`
- Verify: `helm version`

### Core commands (muscle memory)
```bash
helm repo add stable https://charts.helm.sh/stable   # add repo
helm repo update                                     # refresh indexes
helm search repo nginx                               # find charts
helm pull bitnami/nginx --untar                       # download chart
helm install myapp ./chart                            # install from local dir
helm upgrade myapp ./chart -f values-prod.yaml        # upgrade with values
helm uninstall myapp                                  # remove release
helm list                                             # list releases
helm status myapp                                     # release status
helm get values myapp -o yaml                         # show rendered values
helm template ./chart -f values.yaml                  # render locally (no cluster)
```

### Chart structure (generated by `helm create mychart`)
```
mychart/
  Chart.yaml        # name, version, description, dependencies
  values.yaml       # default values
  templates/        # yaml templates (.tpl)
  charts/           # subcharts
  .helmignore       # files to ignore
```

### Render-first habit
- Use `helm template` to render manifests locally before installing.
- Combine with `kubectl apply -f -` for dry-run in pipelines: `helm template ... | kubectl apply --dry-run=client -f -`

### Repos and provenance
- Charts live in HTTP(S) repos (index.yaml). Add/remove with `helm repo add|remove`.
- For trusted sources, use chart signing (`helm show chart` to inspect, `helm verify` with provenance).

### Quick sandbox exercise
```bash
helm create demo
cd demo
helm template . | head -40            # inspect rendered manifests
helm install demo .                   # install to current kube-context
helm uninstall demo
```
